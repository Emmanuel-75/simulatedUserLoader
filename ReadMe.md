# Simulated User Loader

This project focuses on mastering asynchronous JavaScript by building a simulated user loader that demonstrates callbacks vs. promises, including handling nested callbacks (“callback hell”), promise chaining, error handling, and simulating delayed API calls with setTimeout().

## Features

- Callbacks & Promises
  Toggle between using traditional callbacks and modern promises to fetch simulated user data and posts.
- Simulated API Calls
  Uses setTimeout() to mimic network delays for fetching user info and posts.
- Error Handling
  Randomized network failure simulation to practice .catch() and error callbacks.
- Dynamic Data Rendering
  User names, posts, and likes are dynamically rendered into the DOM.
- Loading Animations
  Tailwind CSS transitions show a spinning loader while fetching data.
- Responsive Tailwind Card Layout
  Clean UI with gradient backgrounds, rounded cards, and flexible layout.
- Semantic HTML Structure
  Organized for readability and maintainability.

## Tech Stack

- HTML5
- Tailwind CSS
- JavaScript (Vanilla JS)

## Purpose

This project reinforces key frontend engineering concepts:

- Handling asynchronous operations with callbacks and promises
- Avoiding callback hell and improving code readability with promise chaining
- Implementing error handling for network-like operations
- Dynamically rendering UI based on JavaScript data structures
- Structuring code for clear separation between logic and UI
